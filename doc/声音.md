## 一、概述

MiniMax Speech-02-HD 是本项目使用的核心语音合成（TTS）服务，通过 WaveSpeed AI 平台统一调用。该服务支持高清音质、多种语音角色、情绪控制、停顿调节等高级功能，为数字人视频提供自然流畅的配音。

### 1.1 核心特性

- **高清音质**: 支持 32kHz 采样率，192kbps 比特率
- **150+ 声音角色**: 涵盖中英文、粤语、台湾国语及 30+ 种语言
- **7 种情绪控制**: neutral/happy/sad/angry/fearful/disgusted/surprised
- **毫秒级停顿控制**: 精确到 0.01 秒的停顿标记
- **智能英文规范化**: 自动处理数字、日期、缩写等特殊表达
- **自定义声音**: 通过 Voice Design API 基于文本描述创建个性化声音

### 1.2 在数字人流程中的位置

```
用户输入文本脚本
        ↓
阶段 1: Seedream 生成头像图像
        ↓
阶段 2: MiniMax 生成配音音频 ← 本文档重点
        ↓
阶段 3: Infinitetalk 唇同步视频
        ↓
发布可播放的数字人视频
```

## 二、API 参数详解

### 2.1 基础参数

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| `text` | string | **必填** | ≤5000 字符 | 要合成的文本内容 |
| `voice_id` | string | "male-qn-qingse" | 见官方列表 | 声音角色 ID |
| `speed` | float | 1.0 | 0.5-2.0 | 语速倍数 |
| `pitch` | integer | 0 | -12 至 12 | 音调偏移（半音） |
| `emotion` | string | "neutral" | 见情绪列表 | 情绪类型 |
| `sample_rate` | integer | 32000 | 16000/24000/32000/48000 | 音频采样率（Hz） |
| `channel` | integer | 1 | 1 或 2 | 声道数（1=单声道，2=立体声） |
| `bitrate` | integer | 128000 | 64000-320000 | 音频比特率（bps） |
| `english_normalization` | boolean | true | true/false | 英文数字智能发音 |

### 2.2 情绪参数（Emotion）

| 情绪值 | 英文 | 说明 | 适用场景 |
|--------|------|------|----------|
| `neutral` | 中性 | 平稳叙述，无明显情绪倾向 | 新闻播报、正式通知 |
| `happy` | 快乐 | 语调上扬，节奏轻快 | 营销推广、庆祝内容 |
| `sad` | 悲伤 | 语调低沉，节奏缓慢 | 悼念视频、情感故事 |
| `angry` | 愤怒 | 语调强硬，音量加大 | 批评性评论、辩论内容 |
| `fearful` | 恐惧 | 语调紧张，音调不稳 | 悬疑剧情、警示内容 |
| `disgusted` | 厌恶 | 语调排斥，带有否定感 | 批判性内容、负面评价 |
| `surprised` | 惊讶 | 语调突然上扬，带有疑问感 | 揭秘内容、惊喜公告 |

### 2.3 停顿控制语法

**语法**: `<#停顿时长#>`，时长范围：0.01-99.99 秒

**示例**:
```python
text = """
大家好，我是 AI 主播小智。<#0.5#>
今天为您带来三条重要新闻。<#0.8#>
第一<#0.3#>，科技领域传来重大突破。<#0.6#>
感谢收看，我们下期再见。
"""
```

**最佳实践停顿时长**:
- 逗号：0.3-0.5 秒
- 句号：0.6-0.8 秒
- 段落间：1.0-1.5 秒
- 强调停顿：1.5-2.5 秒
- 思考停顿：2.5-4.0 秒

## 三、服务实现（Python）

### 3.1 核心类：MiniMaxTTSService

**文件位置**: `py/services/minimax_tts_service.py`

```python
class MiniMaxTTSService:
    """MiniMax speech-02-hd TTS 客户端"""

    def __init__(self, api_key: str):
        """
        初始化 TTS 服务

        Args:
            api_key: MiniMax API 密钥（或 WaveSpeed API 密钥）
        """
        self.api_key = api_key
        self.base_url = "https://api.wavespeed.ai/api/v3"
        self.endpoint = f"{self.base_url}/minimax/speech-02-hd"
        self._poll_interval = 3
        self._poll_timeout = 180

    async def generate_voice(
        self,
        text: str,
        voice_id: str = "male-qn-qingse",
        speed: float = 1.0,
        pitch: int = 0,
        emotion: str = "neutral",
        output_path: Optional[Path] = None
    ) -> Dict[str, Any]:
        """
        生成语音

        Args:
            text: 文本内容
            voice_id: 音色 ID
            speed: 语速 0.5-2.0
            pitch: 音调 -12~12
            emotion: 情绪 neutral/happy/sad/angry
            output_path: 输出路径（可选）

        Returns:
            {
                "audio_url": "https://...",
                "audio_path": "output/.../speech.mp3",  # 如果提供了 output_path
                "duration": 12.5,
                "cost": 0.025
            }

        Raises:
            ExternalAPIError: 外部 API 调用错误
        """
        # 1. 构建请求参数
        payload = {
            "text": text,
            "voice_id": voice_id,
            "speed": speed,
            "pitch": pitch,
            "emotion": emotion,
            "sample_rate": 32000,
            "channel": 1,
            "english_normalization": True
        }

        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }

        # 2. 发送请求并重试
        result = await self._with_retry(lambda: self._request_tts(payload, headers))
        audio_url, duration = await self._resolve_audio_output(result, headers)

        # 3. 下载音频（如果提供了输出路径）
        if output_path:
            await self._download_audio(audio_url, output_path)

        # 4. 计算成本
        cost = self._calculate_cost(duration if duration else 0.0)

        return {
            "audio_url": audio_url,
            "audio_path": str(output_path) if output_path else None,
            "duration": duration,
            "cost": cost
        }
```

### 3.2 重试机制

```python
async def _with_retry(
    self,
    func: Callable[[], Any],
    max_attempts: int = 3,
    base_delay: int = 5
):
    """
    指数退避重试策略

    - 第 1 次重试：等待 5 秒
    - 第 2 次重试：等待 10 秒
    - 第 3 次重试：等待 15 秒
    """
    attempt = 0
    while True:
        attempt += 1
        try:
            return await func()
        except ExternalAPIError as err:
            if not err.is_retryable() or attempt >= max_attempts:
                raise
            await asyncio.sleep(base_delay * attempt)
```

### 3.3 成本计算

```python
def _calculate_cost(self, duration: float) -> float:
    """
    计算成本

    Args:
        duration: 音频时长（秒）

    Returns:
        成本（美元）

    规则:
        $0.02/分钟
    """
    return (duration / 60) * 0.02
```

## 八、最佳实践

### 8.1 语音自然度优化

1. **选择匹配的 voice_id**
   - 数字人性别/年龄与 voice_id 一致
   - 商务内容用 `Professional_Female/Male`
   - 娱乐内容用 `Lively_Girl/Young_Male`

2. **语速建议**
   - 新闻播报：1.0（标准）
   - 教育培训：0.9-1.0（稍慢清晰）
   - 营销推广：1.1-1.2（活力感）
   - **避免**: 超过 1.3 会显得急促

3. **音调微调**
   - 微调范围：-3 至 +3（自然可接受）
   - 极端值（±10 以上）会产生"机器人感"

4. **标点符号控制节奏**
   - 使用逗号、句号自动产生自然停顿
   - 只在需要强调时使用 `<#停顿#>` 标记
   - 避免每句话超过 2-3 个停顿标记

### 8.2 情绪与场景匹配

| 场景类型 | 推荐情绪 | 语速 | 音调 |
|---------|---------|------|------|
| 新闻播报 | neutral | 1.0 | 0 |
| 营销推广 | happy | 1.1 | +2 |
| 教育培训 | neutral | 0.9 | 0 |
| 悼念追思 | sad | 0.85 | -3 |
| 批评评论 | angry | 1.0 | 0 |
| 惊喜揭晓 | surprised | 1.05 | +1 |

### 8.3 英文规范化场景

**建议启用** `english_normalization: true` 的场景：
- 中英文混合文本
- 包含数字、日期、时间
- 技术文档（代码、编号）
- 金额、单位表达

**示例转换**:
| 原始文本 | 启用后读法 |
|----------|----------|
| "Dr. Smith" | "Doctor Smith" |
| "2025-12-31" | "二零二五年十二月三十一日" |
| "$99.99" | "九十九点九九美元" |
| "3rd place" | "第三名" |

### 8.4 音质参数配置

| 使用场景 | 采样率 | 比特率 | 说明 |
|---------|--------|--------|------|
| 新闻/教育 | 32000Hz | 192000bps | 高清标准 |
| 营销/娱乐 | 32000Hz | 256000bps | 最丰富音质 |

## 九、高级功能

### 9.1 Voice Design 自定义声音

**API 端点**: `https://api.wavespeed.ai/api/v3/minimax/voice-design`

**用途**: 通过文本描述创建个性化声音（无需音频样本）

**示例**:
```python
# 创建自定义声音
custom_voice_payload = {
    "description": "深沉有力的男性声音，带有轻微的磁性，语调稳重可靠，适合播报财经新闻",
    "language": "zh"
}

# 调用 Voice Design API
response = await service.create_custom_voice(custom_voice_payload)
custom_voice_id = response["voice_id"]

# 使用自定义声音生成语音
result = await service.generate_voice(
    text="今日股市收盘报告...",
    voice_id=custom_voice_id,
    emotion="neutral"
)
```

**注意事项**:
- 自定义声音必须配合 speech-02-hd 模型使用至少一次才能永久保存
- 未使用的自定义声音 7 天后自动删除
- 不支持基于音频样本的声音克隆（法律合规限制）

### 9.2 情绪+语速+音调的三维配合

**组合公式**:
```
真实效果 = 情绪基调 × 语速倍数 × 音调偏移
```

**经典组合**:
1. **活力四射**
   - `emotion: "happy"` + `speed: 1.05` + `pitch: +2`

2. **沉重悲伤**
   - `emotion: "sad"` + `speed: 0.85` + `pitch: -3`

3. **专业播报**
   - `emotion: "neutral"` + `speed: 1.0` + `pitch: 0`

4. **激情演讲**
   - `emotion: "happy"` + `speed: 1.1` + `pitch: +1`

5. **恐怖氛围**
   - `emotion: "fearful"` + `speed: 0.9` + `pitch: -2`

### 9.3 综合案例：打造超真实配音

**场景**: 科技产品发布会主持人


## 十、故障排查

### 10.1 常见问题

**问题 1: API 返回 401 Unauthorized**
- **原因**: API 密钥无效或过期
- **解决**: 检查 `.env` 中的 `MINIMAX_API_KEY`

**问题 2: 生成的语音听起来不自然**
- **原因**: 参数配置不当
- **解决**:
  - 语速控制在 0.9-1.1 之间
  - 音调控制在 -3 至 +3 之间
  - 减少停顿标记使用频率

**问题 3: 音频时长与预期不符**
- **原因**: 停顿标记过多或语速设置错误
- **解决**:
  - 检查文本中的 `<#停顿#>` 标记总时长
  - 实际时长 ≈ 文本字数 / (速度 × 4.0 字/秒) + 停顿总时长

**问题 4: 成本超出预期**
- **原因**: 文本过长或频繁调用
- **解决**:
  - 限制单次文本 ≤ 500 字符（约 2 分钟）
  - 调试时使用更短文本
  - 查看 `task.json` 中的成本估算字段

### 10.2 日志分析

**日志位置**: `output/<job_id>/log.txt`

**关键标记**:

### 10.3 性能优化

**优化速度**:
- 使用并发调用（如果生成多个任务）
- 降低采样率至 24000Hz（调试阶段）

**优化成本**:
- 限制文本长度在必要范围内
- 复用常用配音（缓存音频文件）
- 批量任务申请企业折扣

---

## 十一、局限性与未来方向

### 11.1 当前技术局限

1. **无 SSML 支持**: 不支持语音合成标记语言（Speech Synthesis Markup Language），无法在单句中实现复杂韵律控制

2. **情绪单一性**: 每段文本只能应用一种 emotion，无法动态切换

3. **停顿精度依赖**: 虽然可以精确到 0.01 秒，但实际效果受模型渲染影响，可能有 ±0.1 秒误差

4. **声音克隆限制**: Voice Design 基于文本描述而非音频样本，无法完全复制特定人声

5. **无名人声音**: 不支持预设的名人或 IP 角色声音（法律合规限制）

### 11.2 法律与伦理考量

**未经授权不得**:
- 克隆名人声音（侵犯肖像权和声音权）
- 使用受版权保护的 IP 角色声音（如动漫角色）
- 制作深度伪造内容（多国立法限制）

**合法替代方案**:
1. 使用 Voice Design 创建风格相似但不侵权的声音
2. 雇佣配音演员模仿特定风格
3. 获取 IP 所有方正式授权（成本高昂）

### 11.3 未来改进方向

**技术演进预测**（基于行业趋势）:

1. **动态情绪控制**: 单段文本中支持情绪标记
   ```
   "我很高兴<emotion:happy>宣布</emotion>，但也<emotion:sad>不舍</emotion>"
   ```

2. **SSML 标准支持**: 引入 W3C SSML 标准，精细韵律控制

3. **基于音频的声音克隆**: 上传音频样本生成自定义声音（需授权审核）

4. **实时情感检测**: AI 自动分析文本语义，智能推荐最佳情绪组合

5. **多角色对话优化**: 单段文本支持多个 voice_id，自然对话场景

```bash
    "pitch": 0,
    "emotion": "neutral",
    "sample_rate": 32000,
    "channel": 1,
    "bitrate": 128000,
    "english_normalization": true
  }'
```

### B2. Python 完整示例

```python
import asyncio
from pathlib import Path
from py.services.minimax_tts_service import MiniMaxTTSService

async def main():
    # 初始化服务
    service = MiniMaxTTSService(
        api_key="your_api_key_here"
    )

    # 生成语音
    result = await service.generate_voice(
        text="""
        大家好，我是你们的 AI 主播。<#0.5#>
        今天为大家带来最新的科技资讯。<#0.8#>
        感谢收看！
        """,
        voice_id="Wise_Woman",
        speed=1.0,
        pitch=0,
        emotion="neutral",
        output_path=Path("output/test/speech.mp3")
    )

    # 打印结果
    print(f"✅ 语音生成成功！")
    print(f"   音频 URL: {result['audio_url']}")
    print(f"   本地路径: {result['audio_path']}")
    print(f"   时长: {result['duration']:.2f} 秒")
    print(f"   成本: ${result['cost']:.4f}")

# 运行
asyncio.run(main())
```
