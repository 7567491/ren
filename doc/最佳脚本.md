## 核心设计原则

### 目标受众画像
- **决策层**：CTO/CEO（关注ROI、战略价值）
- **执行层**：IT经理/技术负责人（关注成本、效率、可靠性）
- **使用层**：开发者/数据科学家（关注技术能力、易用性）

### 内容结构（基于最佳实践）
```
开场钩子（3秒法则）→ 核心价值 → 场景证明 → CTA
```

### 信息密度控制
- **每镜头**：1句核心信息（8-15字旁白）
- **字幕规范**：≤18字/行，双行字幕≤36字
- **节奏**：2-4秒/镜头（云计算/AI广告偏快节奏）
- **总时长**：3镜9-15s / 6镜18-30s / 9镜27-45s

### 品牌护栏（示例）
- **禁用词**：绝对领先、唯一、100%、永久免费
- **必须元素**：Logo（右下角）、企业备案号、联系方式
- **色彩基调**：科技蓝#0066FF、智能紫#6C5CE7、数据绿#00B894

## 统一运镜模式系统

**6种基础运镜**（适配所有脚本类型）

| 运镜代号 | 名称 | 描述 | 适用场景 | 时长 |
|---------|------|------|---------|------|
| **M1** | 推进特写 | 从中景推进到产品/界面特写 | 开场钩子、核心功能强调 | 3-4s |
| **M2** | 俯视展开 | 上帝视角俯瞰，展现宏观架构 | 云架构、数据中心、全局视角 | 3-5s |
| **M3** | 平移扫过 | 水平移动展示多个场景/应用 | 多场景应用、功能列表 | 4-5s |
| **M4** | 旋转环绕 | 360度围绕主体旋转展示 | 产品展示、AI模型可视化 | 3-4s |
| **M5** | 拉出揭示 | 从细节拉出到全景 | 从单点到整体价值 | 4-5s |
| **M6** | 分屏对比 | 左右/上下分屏并列展示 | Before/After、多场景并行 | 3-4s |

**运镜组合规则**
- 开场必用：M1（推进特写）吸引注意力
- 中段展示：M2/M3/M4组合展现能力
- 对比证明：M6（分屏对比）展示效果
- 结尾升华：M5（拉出揭示）+ CTA

### 5. 常见问题

**Q1：如何确保视觉风格一致？**
- 固定seed值
- 使用统一的color_scheme配置
- 所有镜头使用相同的style_preset

**Q2：如何提升画面质量？**
- 使用详细的scene_description（包含光线、色彩、材质）
- 参考成功案例，使用"cinematic lighting"等关键词
- 避免复杂运动（motion_strength不超过8）

**Q3：如何优化字幕可读性？**
- 字幕≤18字/行，双行≤36字
- 使用高对比度配色（白字+黑描边 或 黑字+白底）
- 字幕位置：下方1/3处，避开人脸

**Q4：如何加快生成速度？**
- 图像生成：并发2线程（已优化）
- 视频生成：串行处理（避免API限流）
- 使用断点续传：`--resume aka-xxxxx`

**Q5：数据来源如何追溯？**
- 在shots_script.json中添加data_source字段
- 关键数据需要有第三方报告支撑（如IDC、Gartner）
- 避免使用"最佳""第一"等绝对化表述
