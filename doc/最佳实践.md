# AI 广告视频自动生成工作流最佳实践

## 总览
- 目标：低成本、可量产、可追溯的商业视频素材；全流程可审计和可回滚。
- 基线流程：需求对齐 → 文案/分镜 → 资产生成 → 音频/字幕 → 合成 → 审核 → 分发/监测。
- 重要原则：前置品牌与法务要求、提示词标准化、版本可追踪、A/B 持续优化。

## 需求与策略
- 受众与渠道：明确投放平台、尺寸比例、时长（常见 6-30s）和首屏 3 秒吸引点。
- 品牌护栏：列出禁用词/敏感场景、Logo 使用规范、色板/字体/语气基调。
- 行动目标：1 个核心 CTA，脚本按「开场钩子-价值-证明-CTA」结构编排。
- 数据资产：准备示例优质广告、成功脚本，作为少样本参考或图像参考帧。

## 提示与脚本
- 模板化提示：固定分镜数量、比例、情绪、镜头运动，避免模型自由发挥导致风格漂移。
- 信息密度控制：每镜头一句关键信息，字幕≤18 字/行；文案先行再适配口播时长。
- 多版本生成：一次产出 3-5 个脚本版本；记录参数（种子、风格、模型版本）便于复现。
- 安全过滤：提示词内预置品牌禁用词检测，失败时自动回退或重写。

## 视觉与视频生成
- 模型选择：图像用稳定风格模型，视频用 I2V 串行生成，减少质量波动与限流风险。
- 一致性控制：固定角色/场景参考帧，使用相同风格描述、色板、光照与镜头运动。
- 画面节奏：广告更偏快节奏，镜头 2-4s；避免复杂运动导致口型/字幕错位。
- 失败重试：设置可重试异常与指数退避；中间产物（图/静音视频）落盘，便于断点续跑。

## 配音与字幕
- 先文稿后配音：基于定稿脚本生成 TTS；选择贴合品牌的音色与语速（一般 0.9-1.05x）。
- 时间对齐：使用强制对齐或字幕工具生成带时间戳 SRT，再合成到视频。
- 多语言考虑：本地化时单独重写 Hook 与 CTA，而不是直接翻译。

## 合成与导出
- 音频优先：先将配音与视频对齐，再叠加 BGM，控制对话-音乐在 -6~-3 dB 与 -20~-14 dB。
- 字幕样式：保证读屏安全色与对比度，使用品牌色但避免纯白背景。
- 导出规范：按渠道需要生成多尺寸（竖版 9:16、方形 1:1、横版 16:9），并压缩到平台友好码率。

## 审核与评估
- 技术检查：无水印/错别字/卡顿/音画不同步；关键帧无畸变或多手指等常见 AI 伪影。
- 品牌与法务：确保无夸张疗效、竞品贬损、违规宣称；人物/音乐素材版权可追溯。
- 可观测性：保存生成日志（提示词、参数、模型版本、耗时、成本）；为每条素材加 UTM。

## 实验与迭代
- A/B 机制：一次上线 2-3 个版本，控制变量（画面/文案/CTA）单一化。
- 指标闭环：按渠道查看 CTR、CVR、CPA、前 3 秒完播率，结合评论反馈迭代脚本与画面。
- 成本控制：先用低分辨率/少镜头做验证，确认概念后再升分辨率与镜头数。

## 工具与自动化建议
- 脚本与分镜：对话模型（如 DeepSeek）+ 固定提示模板，输出 JSON 结构便于后续流水线。
- 配音与字幕：TTS（如 Edge TTS）+ 自动对齐工具，生成 mp3 与 SRT。
- 图像/视频：文本转图 + I2V（如 WavespeedAI），采用串行视频生成与并发图生图。
- 合成：使用 MoviePy 或 FFmpeg 批量化；所有中间文件命名有序（shot_X_*），便于断点续跑与溯源。
